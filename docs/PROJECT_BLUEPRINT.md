# 项目总方案

## 产品定位

科大 Agent 不是“自动生成创新点或整篇论文”的工具，而是带证据的研究机会发现与论文逆向工程助手。它服务于三段任务：读懂论文、看清方向、推进自己的研究。

## 核心能力

1. 论文检索与代表工作筛选。
2. 单篇论文叙事链拆解。
3. Claim—Experiment—Artifact—Evidence 对齐。
4. 实验意图与图表角色解释。
5. 多论文方法、数据集和指标比较。
6. 有支持/反对证据、覆盖范围和置信度的研究机会候选。
7. 从用户研究假设反推实验与图表需求。

## 数据分层

| 层级 | 建议规模 | 深度 |
|---|---:|---|
| Bronze | 500–1000 | 元数据、摘要、会刊、年份、主题 |
| Silver | 200–300 | 章节、方法、数据集、指标、图表自动提取 |
| Gold | 30–50 | 人工复核叙事链、实验意图、图表角色、证据锚点 |

比赛版先完成 5–10 篇 TAD 论文的高质量闭环。

## 技术结构

- 前端：Vue 3 + TypeScript + Vite。
- API：FastAPI + Pydantic。
- 权威结构化存储：MySQL。
- 向量检索：Milvus。
- 关系索引：Neo4j。
- PDF：统一契约，适配 PyMuPDF、GROBID、MinerU。
- 智能体：讯飞星辰作为可配置网关，本地模式可独立运行。
