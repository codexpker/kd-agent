# 星辰工作流节点设计 v1

## P0-X1：单篇论文最小真实链路

| 顺序 | 节点 | 输入 | 输出/门禁 |
| --- | --- | --- | --- |
| 1 | 用户输入 | 自然语言问题 | 保存会话ID与原始任务 |
| 2 | 意图分类 | 用户问题、历史 | 三工作台之一；低置信度进入追问 |
| 3 | 阅读目标追问 | 论文、目标缺失项 | 快速理解/实验复现/研究设计 |
| 4 | 论文检索 | 主题或标题 | 调用`search_papers`，展示标注状态 |
| 5 | 单篇拆解 | `paper_id`、`focus` | 调用`deconstruct_paper` |
| 6 | 证据门禁 | 工具统一响应 | `insufficient_evidence`停止；`partial`附边界 |
| 7 | 星火组织 | 工具结果、系统提示词 | 只组织已有事实并引用EvidenceAnchor |
| 8 | 输出审计 | 回答、工具响应 | 拒绝未知EvidenceAnchor；记录trace和版本 |
| 9 | Vue回跳 | EvidenceAnchor ID | 打开论文证据详情或明确无授权页图 |

## 后续分支

- 领域进展地图：检索 → 用户确认论文范围 → `compare_papers` → 覆盖门禁 → 候选机会审计。
- 我的研究教练：收集 Claim 四字段 → `diagnose_claim` → 用户确认/修改 → 实验与图表计划。
- MaaS：只接入“TAD实验协议风险诊断”，在 Claim/实验计划确认后调用；不得替代证据工具。

## 正式发布门禁

- 将 `agent-tools.openapi.yaml` 中服务器地址替换为真实 HTTPS 地址。
- 部署环境设为非 `local`，并在星辰与服务端配置同一 Bearer Token。
- 保存发布后的 Flow ID、工作流版本、星火模型名称/版本和不含密钥的截图。
- 运行成功、证据不足、鉴权失败、超时、工具失败五条路径；保存脱敏请求ID和延迟。
- 在完成 MaaS ServiceID 与盲测前后对比前，不得将完整讯飞主链路标记为已核验。
