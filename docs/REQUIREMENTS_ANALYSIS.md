## 3. 赛事要求解读

### 3.1 必须满足的作品边界

赛事要求作品解决一个或多个真实教学或科研场景需求，并推荐使用学科知识库、模型微调、
RAG、智能工作流、多模态和场景化开发。本项目选择真实科研训练场景：人工智能科研新手
需要读懂论文论证、掌握领域进展，并把方法迁移到自己的选题、实验和写作中。

赛事展示的科研辅助能力包括前沿追踪、学术写作、科研数据分析与可视化。项目保留这些
全流程能力，但用“论文逆向工程”把它们连成一条有差异化、可深度验收的主线，而不是做
彼此孤立的通用问答按钮。

### 3.2 核心功能要求

| 赛事编号 | 原要求归纳 | 本项目验收口径 |
| --- | --- | --- |
| C1 | 深度贴合场景 | 从真实论文问题出发完成检索、逆向工程、比较、实验/图表规划闭环 |
| C2 | 多轮自然交互 | 能识别阅读目标，缺少论文、领域、时间或研究假设时主动澄清 |
| C3 | 知识权威可信 | 核心论文来自权威会刊/开放来源，字段有来源和审核状态 |
| C4 | 内容可追溯 | 关键解释能定位论文、章节、页码或Figure/Table，找不到时明确证据不足 |
| I5 | 可复制与推广 | 配置新AI子领域的来源注册表和标注规范即可扩展，完成初步用户验证 |
| I6 | 内容多样性 | 同时处理文本、公式说明、图、表、引用关系和结构化实验结果 |

## 4. 产品定位

### 4.1 一句话定义

AI-Research Copilot 是面向人工智能科研新手的证据驱动科研辅助智能体：以论文逆向工程
为核心差异化入口，帮助用户理解优秀论文如何提出问题、设计方法、组织实验和图表、形成
可核验结论，并进一步完成前沿追踪、选题分析、实验设计、写作脚手架和成果总结。

### 4.2 不是简单PDF聊天

系统的最小价值单位不是一段自然语言回答，而是一个带证据的科研决策对象：

```text
Problem → Gap → Hypothesis → Method → Claim
        → Experiment → Figure/Table → Evidence → Boundary
```

PDF问答、摘要和术语解释是必要的基础体验，但不能作为本项目的唯一创新点。

### 4.3 能力分层

| 层级 | 能力 | 比赛版优先级 |
| --- | --- | --- |
| 核心差异化 | 单篇论文叙事逆向工程、实验意图、图表角色、多论文论证比较 | P0 |
| 全流程承接 | 前沿追踪、选题机会审查、实验与图表计划、写作脚手架、成果总结 | P0/P1 |
| 基础体验 | PDF上下文问答、术语/公式/图表解释、论文检索、引用跳转 | P0 |
| 平台能力 | 多轮规划、个性化、来源核验、知识图谱、评测与审计 | P0 |
| 赛后扩展 | 自动实验执行、大规模跨学科语料、团队协作 | P2 |

## 5. 用户与真实任务

### 5.1 主要用户画像

**P1：科研入门学生。** 人工智能专业研一或高年级本科生，知道基础模型但不了解顶会论文
叙事和实验惯例，需要在有限时间内完成组会、复现、开题和课程研究。

**P2：正在做课题的研究生。** 已有问题和初步方案，但不确定现有工作进展、创新切入点、
实验是否足够、图表能否支持结论。

**P3：指导教师/博士生。** 希望学生先形成结构化理解和可检查的论证链，减少重复讲解，
同时不能让AI替代学术判断。

### 5.2 关键Job-to-be-Done

1. 组会前，在30分钟内理解论文的问题、方法、实验和局限，并能指出证据位置。
2. 进入陌生方向时，快速知道代表工作、方法演进、常用数据集与当前争议。
3. 阅读实验部分时，理解每项实验为什么存在、控制了什么、能够和不能够证明什么。
4. 阅读Figure/Table时，理解为何选用这种表达形式、阅读顺序及其叙事作用。
5. 有自己的想法后，检查Claim需要哪些基线、消融、鲁棒性和图表证据。
6. 写论文时获得结构和论证脚手架，而不是复制原文或代写不可核验内容。
7. 汇总结果时，根据真实实验数据生成可执行绘图代码并检查结论一致性。

## 6. 竞品能力吸收与差异化

### 6.1 SciSpace

SciSpace Copilot的成熟之处是围绕PDF上下文即时解释文本、数学内容、表格、图示和术语，
降低论文阅读的切换成本。项目吸收“选中即解释、问题与证据同屏、图表就地问答”的交互，
但不复制其通用PDF聊天定位。参考：
[SciSpace Copilot介绍](https://scispace.com/resources/introducing-copilot-ai-assistant-explains-research-papers/)、
[SciSpace Search](https://scispace.com/search)。

### 6.2 Semantic Scholar / Semantic Reader

Semantic Reader强调结构感知阅读、上下文浮层、引用卡片和用户可控的阅读增强；TLDR提供
极短论文摘要；Paper Finder通过迭代搜索、追踪引用和相关性说明支持文献发现。项目吸收
渐进式略读、引用上下文、相关性解释和可控高亮，不把所有模型判断强加给用户。参考：
[Semantic Reader](https://www.semanticscholar.org/product/semantic-reader)、
[Semantic Scholar TLDR](https://www.semanticscholar.org/product/tldr)、
[Paper Finder](https://allenai.org/blog/paper-finder)、
[OpenReader工具链](https://openreader.semanticscholar.org/)。

### 6.3 本项目的差异化

| 维度 | 成熟产品可借鉴点 | 本项目必须额外做到 |
| --- | --- | --- |
| PDF阅读 | 上下文解释、选中文本提问 | 从章节和图表还原完整科研叙事链 |
| 论文检索 | 相关论文、引用网络、简短摘要 | 显示检索计划、入选原因和语料覆盖边界 |
| 图表理解 | 解释图表内容 | 回答“为什么此处用这张图/表、支持哪个Claim、不能证明什么” |
| 实验理解 | 总结实验结果 | 结构化Research Question、假设、控制变量、证据角色和边界 |
| 写作辅助 | 改写、语言润色 | 基于用户真实证据生成论证脚手架并检查Claim—Evidence一致性 |
| 培养价值 | 更快读完论文 | 让学生逐步学会自己拆解论文和设计实验，可测量能力迁移 |

项目不得照搬竞品界面、文案、数据或实现；竞品能力只作为需求基准和交互启发。

| NFR-006 | 隐私 | 用户测试数据最小化和脱敏 | 使用匿名ID，原始身份映射不进仓库和模型训练集 |
| NFR-007 | 合规 | 论文全文与图表按许可处理 | 记录license与access_policy；默认不再分发来源不清内容 |
| NFR-008 | 可维护 | 三库职责清晰，索引可重建 | MySQL为权威库；Milvus/Neo4j带corpus version |
| NFR-009 | 可扩展 | 可通过领域配置扩展到其他AI方向 | 来源注册表、标签体系和Gold规范不硬编码TAD |
| NFR-010 | 可用性 | 学生不阅读手册也能完成黄金任务 | 5名形成性测试中≥4名无需协助完成核心流程 |

## 9. 数据需求

### 9.1 数据分层

| 层级 | 规模目标 | 数据内容 | 用途 |
| --- | ---: | --- | --- |
| Bronze | 500—1000篇 | 元数据、摘要、来源、会刊、年份、主题、许可状态 | 检索与前沿覆盖 |
| Silver | 200—300篇 | 合法全文派生结构、章节、方法、数据集、指标、图表元信息 | 自动抽取与多论文比较 |
| Gold | 比赛前30—50篇；首闭环5—10篇 | 双人复核叙事动作、Claim、实验意图、图表角色、证据锚点 | 评测、演示和MaaS训练/验证 |

论文按统一`paper_id`入库，不按会刊拆成互相隔离的数据表。会刊、届次、来源和许可通过
注册表字段维护，从而既能按顶会/顶刊筛选，又能去重和跨来源更新。

### 9.2 来源策略

- 优先官方会刊元数据、OpenAlex、Crossref、OpenReview、arXiv等稳定来源。
- 全文仅处理开放许可、作者上传或用户合法提供的内容；记录许可与抓取版本。
- 自动抽取是派生数据，必须保存生成模型、提示词、证据位置和审核状态。
- 论文、图表和参考文献不得凭模型补全；无法核验时返回缺失。

## 10. 评测与验收需求

### 10.1 至少三个典型问题

| 用例 | 用户问题 | 正确性证明 |
| --- | --- | --- |
| TC-01 单篇逆向工程 | Anomaly Transformer为什么提出关联差异，哪些实验支持它？ | Gold卡片逐项对照论文章节、图表和专家判断 |
| TC-02 图表透镜 | 指定主结果表、消融表或ROC图为何这样设计，能证明什么？ | 图表—Claim关联准确率、证据位置和边界复核 |
| TC-03 研究教练 | 给定一个TAD假设，应该补哪些实验和图表？ | 教师/博士生盲评完整性与可执行性，并比较普通LLM |
| TC-04 多论文进展 | Transformer类TAD目前工作如何演进、候选切入点在哪里？ | 检索Recall、引用准确率、冲突证据和语料边界检查 |

### 10.2 离线指标

- 检索：Recall@K、MRR、nDCG@K、P50/P95延迟。
- 抽取：Method/Dataset/Metric/Claim/NarrativeMove微平均F1。
- 实验/图表：意图分类F1、Artifact—Claim关系准确率、证据定位准确率。
- 可信度：引用准确率、无证据结论率、人工复核通过率。
- 绘图：代码执行成功率、图表数据一致率。

### 10.3 用户对照实验

50名以上AI专业学生随机或配对完成相同任务，对照条件为“原始PDF + 常用检索/普通大模型”
与“AI-Research Copilot”。记录：正确率、完成时间、证据定位成功率、认知负担、SUS、
继续使用意愿和开放反馈。至少2名真实目标用户形成可提交的身份描述与反馈，教师/博士生
对标准答案和开放任务进行盲评。

## 12. 范围与发布判定

### 12.1 比赛最小可获奖版本

必须同时具备：

1. 一个星辰Agent参与的真实多轮入口。
2. 5—10篇TAD Gold论文的单篇逆向工程。
3. 一个实验意图与图表角色的可视化证据链。
4. 至少3篇论文的比较、进展和候选切入点。
5. 一个用户研究假设到实验/图表计划的教练链路。
6. 至少3个Gold测试问题、2名目标用户和50+学生对照数据。
7. 可访问Demo、可复现代码、合规声明、3分钟视频和量化报告。

### 12.2 非比赛前置项

- 自研通用大模型或全量论文预训练。
- 自动执行所有类型的深度学习实验。
- 覆盖所有人工智能子领域和所有顶刊顶会。
- 完整文献管理器、多人实时协作和商业计费。

这些能力不阻塞比赛版，只有在核心指标已达标且有明确用户收益时才进入开发。

## 13. 风险与待确认事项

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 截止日期口径冲突 | 提交逾期 | 按9月5日倒排，立即在赛事群确认并留存答复 |
| 星辰工具反向调用需要公网HTTPS | 正式编排受阻 | 先采用本地后端出站调用；最终用云端网关/部署，不要求开发机固定公网IP |
| 全文许可不清 | 数据和知识产权风险 | 元数据与派生标注分离，许可白名单，禁止仓库提交受限PDF |
| 模型把推断写成事实 | 内容质量失分 | 证据门控、判断类型标签、Gold评测和人工复核 |

## 14. 需求变更规则

新需求只有在满足以下至少一项时进入比赛版：直接提升某个赛事评分项；补齐提交物；改善
黄金链路的正确率、时间或可用性；降低可信度、合规或部署风险。任何P0变更必须在本文
增加需求ID、验收标准和追踪关系，并同步更新方案设计与路线图。
