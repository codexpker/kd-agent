# 需求分析

## 目标用户与核心问题

主要用户为需要快速进入研究方向的本科生、研究生和新手科研团队。现有工具擅长搜索、摘要和问答，但通常没有把“为什么这样叙述、为什么设计这个实验、这张图支撑什么结论”连接成可审计链路。

## 功能需求

### P0：比赛最小闭环

- 输入研究主题并返回代表论文。
- 选择论文，展示 Problem、Gap、Claim、Experiment、Artifact、Evidence、Boundary。
- 展示实验意图卡和图表角色卡。
- 每个推断可回到证据锚点；未知论文不生成伪 Gold 分析。
- 版面事实与科研语义判断分层。
- 输出研究机会候选及支持/反对证据、覆盖范围、置信度和人工确认项。

### P1：研究推进

- 多论文方法—数据集—指标矩阵。
- 用户研究假设录入。
- 反向生成实验、消融、效率、鲁棒性和图表需求清单。
- 结果数据导入与可执行作图代码。

## 非功能需求

- 可追溯：所有结论引用稳定 EvidenceAnchor。
- 防幻觉：数据缺失时明确返回 unknown/needs-review。
- 版权合规：只有开放全文、用户私有副本或机构授权材料可持久化解析。
- 可离线演示：无三库、无外部模型密钥也可运行 Gold 样例。
- 可替换：解析器、模型和检索后端通过稳定契约隔离。
