## 当前产品主线

产品北极星是“面向 AI 科研新手的论文逆向工程与科研叙事训练”。三个工作台是论文逆向工程、
领域进展地图和我的研究教练。比赛黄金链路是：

`自然语言任务 → 澄清目标 → 检索真实论文 → 单篇逆向工程 → 实验/图表意图解释 → 多篇比较 → 用户 Claim 诊断 → 可追溯结果`

规则允许使用其他工具，但项目已冻结讯飞能力为正式比赛主链路：

`Vue → FastAPI → 星辰 Agent → 星火/科技文献模型、MaaS → 自研结构化工具`

Vue 是正式 Demo 和证据可视化入口，FastAPI 是工具编排、安全和证据门禁层。星辰负责用户意图、任务路由与三工作台
编排；星火/科技文献模型负责科研语言理解与报告组织；MaaS 负责“时间序列异常检测实验协议风险诊断”垂类微调。
自研检索、证据图谱、引用一致性、时序实验和可视化不交给模型自由生成。

只有完成真实联调并保存脱敏证据后，才能宣称使用了相关平台。离线规则只是开发/故障降级，不得冒充正式链路。

## 当前优先级

- P0：满足赛事第 1—4 项基础功能，并用至少 3 个典型问题证明正确性。
- P0：把 1 篇开发种子扩展为 5—10 篇真实、合法、已核验的 TAD Gold 最小闭环。
- P0：联调并发布星辰正式工作流，实现真正自然的多轮任务交互，而不只依赖关键词分流。
- P0：完成星火/科技文献模型真实调用，以及 MaaS 垂类微调的训练集、ServiceID 和前后对比。
- P0：完成至少 2 名真实目标用户的结构化试用和量化验证。
- P1：从创新功能 5—8 中优先做复杂问题求解、内容多元和可复制推广。

在 P0 闭环完成前，暂停扩张 PDF 解析器评测、更多数据库迁移、复杂知识图谱展示、通用权限、
商业计费和与典型问题无关的后台能力。

## 每轮开发规则

1. 每轮只完成一个可验收的最小任务。
2. 先检查现有实现，禁止重复建设。
3. 保留离线演示，但必须明确标记为演示数据或本地规则。
4. 关键结论必须引用稳定来源和 EvidenceAnchor；未知即返回证据不足。
5. 不提交密钥、受限 PDF、个人信息、用户原始身份映射、构建产物或依赖目录。
6. 完成代码、测试和相关文档同步后，再更新 `ROADMAP.md` 状态。
7. 普通工程选择自行判断；只有产品范围、版权、外部付费、密钥、真实用户数据或不可逆操作
   需要询问用户。

## 完成定义

一个任务只有同时满足以下条件才可标记完成：

- 代码路径真实可执行；
- 正向、失败和证据不足路径有测试；
- 前端生产构建通过（如涉及前端）；
- 状态标签区分开发种子、合成数据、自动解析、人工核验和正式结果；
- 文档与真实状态一致；
- 没有秘密、受限全文、个人数据或生成产物进入 Git。

常用验证命令：

```bash
cd backend && python -m pytest
cd frontend && npm run build
cd frontend && npm run test:e2e
```
